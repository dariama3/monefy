## Q&A

1. Як ти структурував frontend і backend?

Я організувала 3 докер контейнери для Laravel, Nuxt та MySQL. Бекенд та фронтенд знаходяться у своїх папках та запускаються з головного docker-compose.yml файлу.
Фронтенд робить запити до бекенду, бекенд валідує та комунікує з базою даних.

2. Які компроміси ти зробив і чому?

- В рамках завдання було опущено авторизацію користувачів, тобто будь-хто може змінити будь-який інвойс.

3. Що б ти покращив у production-версії?

- Інвойс це єдина сутність, для продакшену точно потрібні будуть сутності користувачів та відповідні звʼязки в базі.
- Потрібно додати можливість зміни статусу інвойсу для бухгалтера.
- Додати повноцінну документаці. для всіх API запитів

4. Які UX edge cases ти врахував?

- При випадковому потраплянні на домашню сторінку, користувача перенаправить на список всіх інвойсів.
